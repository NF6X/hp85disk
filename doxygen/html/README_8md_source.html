9 ## HP85 Disk Emulator Copyright &copy; 2014-2017 Mike Gore
10  * See [COPYRIGHT](COPYRIGHT.md) for a full copywrite notice for the project
11 
12  This project emulates five GPIB devices
13  * first SS80 HP9134L disk at 700 for my HP85A (with 85A roms)
14  * first Amigo 9121D disk at 710 for my HP85B (with 85B roms)
15  * second SS80 HP9134L disk at 720 for my HP85A (with 85A roms)
16  * second Amigo 9121D disk at 730 for my HP85B (with 85B roms)
17  * Printer capture at 750 for my HP54645D scope

135 ## AVR ATMEGA1284P pin assignments for HP85 Disk
136  * @see Documents/P85Disk.pdf for a hand drawn diagram
137  * GPIB: Each GPIB pin (8 data and 8 control lines ) attach to CPU via 120 ohm current limit resistor .
138  * Each GPIB connector pin (8 data and 8 control lines) have a 10K pull-up resistor to VCC.
139  * ISP header: MOSI,MISO,SCK,/Reset connects directly to ISP header
140  * Micro SD Interface: MOSI,MISO,SCK attach to CPU function via a 1k series resistor.
141  * Micro SD interface has level shifters and internal 5V to 3.3V regulator
142  * PB3 /CS must have a 10K pullup up to VCC to prevent access durring ISP programming
143  * PB4 should have a 10K pull up help assure the SPI bus does not go into slave mode.
144  * RS232 TTL: connect to FTDI232 USB board which also provides 5V VCC power to all circuits..
145  * I2C: SCL,SDA connect to optional DS1307 RTC board with each line having a 2k2 pull-up
146 <pre>
147 
148  ATMEGA1284P (and ATMEGA644P)
149  +---V---+
150  5 EOI INT0 PB0 1| |40 PA0 D1 1
151  6 DAV INT1 PB1 2| |39 PA1 D2 2
152  PP INT2 PB2 3| |38 PA2 D3 3
153  SD /CS PWM PB3 4| |37 PA3 D4 4
154  NC PWM PB4 5| |36 PA4 D5 13
155  SD MOSI PB5 6| |35 PA5 D6 14
156  SD MISO PB6 7| |34 PA6 D7 15
157  SD SCK PB7 8| |33 PA7 D8 16
158  10K pullup /RST 9| |32 AREF 0.1uf
159  +5 VCC 10| |31 GND GND

178 ## Parallel Poll Response circuit
179  * Uses: Three chips 74HC05, 74HC32, 74HC595
180  * Parallel Poll Response must be less then 2 Microseconds therefore we use hardware to do it!
181 
182 
183 <pre>
184  ATMEGA HC595 HC05
185  +----V----+ +-V-+
186  3 PB2 -------- 12 |RCLK Q0| 15 -x- 1 | | 2 --- GPIB D8
187  6 MOSI ------- 14 |SER Q1| 1 -x- 3 | | 4 --- GPIB D7
188  8 SCK -------- 11 |SRCLK Q2| 2 -x- 5 | | 6 --- GPIB D6
189  9 IFC -------- 10 |SRCLR Q3| 3 -x- 9 | | 8 --- GPIB D5
190  HC32 | | | | | 7 GND
191  +-V-+ | | | | |14 VCC
192  EOI 2| | | | | +---+
193  ATN 1| | | | \--- each line has its own
194  | | 3--13 |/OE | 10K resistor to GND
195  VCC 14| | | | 16 VCC
196  GND 7| | | | 8 GND
197  +---+ +---------+
198 </pre>
199 
200 Notes: When both EOI and ATN are low the HC32 enables HC595 outputs
201  * If any HC595 output is high the GPIB bus bit will be pulled low
202  * IFC low resets the HC595 outputs low - so the HC05 outputs will float.

206 ## Testing
207  * Testing was done with an HP85A (with extended EMS rom)
208  * Using the Hewlett-Packard Series 80 - PRM-85 by Bill Kotaska
209  * This makes my HP85A look like and HP85B
210  * I can also use the normal mass storage rom if I limit to AMIGO drives.

212  * Note: the EMS rom has extended INITIALIZE attributes
