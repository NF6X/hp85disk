142 ## TELEDISK conversion technical notes - extracting LIF data
143  * Unfortunately TeleDisk saves everything in on all sides and tracks.
144  * Question: what happens when we have a disk formatted multiple times with differing parameters ?
145 
146  * Example 1
147  * Say we start an image that was two sided 80 track disk, 9 sectors, 512 byte sectors
148  * Later on someone formats it single sides 35 track disk, 16 sectors, 256 bytes sectors
149  * Analisis
150  * We examing the first 30 tracks on both sides looking for:
151  * FIRST sector on both sides and its size
152  * LAST sector on both size that matches the size
153  * COUNT of sectors on each side that matches the size
154  * We then compare sector COUNT, compare sector SIZE, and FIRSTd
155  * If SIZE or COUNT is not the same reject
156  * If only FISRT mismatches we then check FIRST for continuation of numbering on each side
157  * Conclusion
158  * We can reject side two by comparing the
159  * FIRST sector number, Number of sectors and size
160 
161  * Eample 2
162  * We have a double sided disk that is formated single sided with FIRST,LAST,COUNT and SIZE matching
163  * Analisis
164  * We have no automated way to reject side two
165  * Conclusion
166  * The user will have to override heads
167 ___

169 ## TELEDISK extracting LIF data user overrides
170  * Number of Sides:
171  * You CAN specify single sided override for two sided disks
172  * (as defined in the TeleDisk headers)
173  * However you can NOT specify 2 sided mode if the TeleDisk image headers say there is one side
174 
175  * Sector Size override
176  * Rather then using the SIZE of the FIRST sector as a filter the user can manually pick the size
177 
178  * Number of tracks overrides
179  * The LIF decoder stops after the last sector of the last file so this option is ignored
180 </pre>

258 ## LIF command help
259  * lif add lifimage lifname from_ascii_file
260  * Add ASCII file converted to E010 format to existing LIF image on SD card
261  * lif add /amigo1.lif TEST1 /test.bas
262  * lif add /amigo1.lif TREK85 /TREK85/TREK85.BAS
263  * Notes:
264  * Strings must be no longer then sector size - 3
265  * Any trailing "\n" and/or "\r" are coverted to "\n" when stored in LIF file
266 
267  * lif addbin lifimage lifname from_lif_image
268  * Adding a binary LIF image files to another LIF image
269  * Examples
270  * lif addbin /amigo1.lif TEST /test.lif
271  * lif addbin /amigo1.lif TREK85 /TREK85/trek.lif
272  * Notes about TREK85 in the examples
273  * Author: TREK85 port was done by Martin Hepperle
274  * https://groups.io/g/hpseries80/topic/star_trek_game_for_hp_85/4845241
275 
276  * lif create lifimage liflabel directory_sectors total_sectors
277  * Create a new LIF image
278  * Example
279  * lif create /amigo3.lif AMIGO3 15 1120
280  * This formats an LIF image file with 15 directory sectors and a length of 1120 (16 * 2 * 35) sectors
281 
282  * lif del lifimage lifname
283  * Delete a file from LIF image on SD card
284  * Example
285  *lif del /amigo1.lif TREK85
286 
287  * lif dir lifimage
